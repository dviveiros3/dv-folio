# Documentation Standards

## File Naming
- `YYYY-MM-DD-Descriptive-Name.md`
- Lowercase, hyphen-separated
- Version: `v1.0` in filename or metadata

## Markdown Standards
- Headers with clear hierarchy (# ## ###)
- Code blocks for all code snippets
- Tables for structured data
- Links to related documents
- Images with alt text

## Required Sections per Document
1. **Title** (H1)
2. **Status**: Draft, Review, Approved, Archived
3. **Owner**: Name or team
4. **Last Updated**: Date
5. **Content**: Clear, scannable, actionable
6. **Related**: Links to related docs

## README Template
```
# Product/Project Name

## Description
One-line description

## Status
Current status and last updated date

## Features
- Feature 1
- Feature 2

## Installation
Step-by-step instructions

## Usage
Basic usage examples

## API Documentation
Link to docs

## Testing
How to run tests

## Roadmap
What's next

## Contributing
How to contribute

## License
License info
```
